# 中国A股股价与股息率监测应用产品需求文档（PRD）

## 文档信息
- **文档版本**: 1.0
- **创建日期**: 2026-04-02
- **产品名称**: 股察 - A股股价与股息率监测平台
- **产品类型**: Web应用
- **目标用户**: 个人投资者

## 1. 产品概述

### 1.1 产品愿景
为个人投资者提供一个简洁、易用的A股股价与股息率监测工具，帮助用户轻松跟踪关注股票的表现，重点关注股息率这一长期投资关键指标。

### 1.2 目标用户画像
- **主要用户**: 个人股票投资者，年龄25-55岁，具备基本股票投资知识
- **投资风格**: 偏向长期投资，关注股息收益
- **技术能力**: 能够熟练使用电脑和智能手机，但不一定是技术专家
- **使用场景**: 日常查看持股表现、筛选高股息股票、跟踪投资组合

### 1.3 核心价值主张
1. **专注股息投资**: 突出股息率计算与分析，满足股息投资者的核心需求
2. **简洁易用**: 现代简约设计，避免传统股票软件的复杂界面
3. **数据本地化**: 无需注册登录，数据本地存储，保护用户隐私
4. **免费数据源**: 基于公开免费数据，降低用户使用成本
5. **离线可用**: 基础功能支持离线使用，数据本地缓存

### 1.4 与竞品差异化
| 特性 | 本产品 | 传统股票软件（同花顺等） | 财经网站（东方财富等） |
|------|--------|--------------------------|------------------------|
| 使用门槛 | 无需注册 | 需要手机号注册 | 部分功能需要注册 |
| 数据存储 | 本地存储 | 云端存储 | 云端存储 |
| 核心焦点 | 股息率监测 | 全面交易功能 | 综合财经资讯 |
| 界面复杂度 | 简约专注 | 复杂全面 | 信息密集 |
| 隐私保护 | 数据不出本地 | 数据上传云端 | 数据上传云端 |

## 2. 功能需求

### 2.1 股票数据展示模块

#### 2.1.1 A股列表浏览
- 功能描述：展示所有A股股票的基本信息列表
- 需求详情：
  - 支持按股票代码、名称搜索
  - 可按股价、涨跌幅、股息率等排序
  - 分页显示，每页显示50条记录
  - 显示字段：代码、名称、当前价、涨跌幅、股息率、市值
  - 支持添加到自选股功能

#### 2.1.2 个股详情页
- 功能描述：展示单只股票的详细信息
- 需求详情：
  - 实时股价显示（日级更新）
  - 当日涨跌幅、振幅、成交量
  - 股息率计算与历史数据
  - 基本面指标：PE、PB、ROE等
  - 公司基本信息：行业、地域、上市日期
  - 添加到自选股按钮

#### 2.1.3 股息率计算与展示
- 功能描述：准确计算并展示股息率信息
- 需求详情：
  - 股息率 = 每股股息 / 当前股价
  - 显示近3年股息支付历史
  - 股息率走势图
  - 与行业平均股息率比较
  - 股息支付日提醒设置

### 2.2 投资组合管理模块

#### 2.2.1 自选股管理
- 功能描述：用户可管理自己关注的股票列表
- 需求详情：
  - 添加/删除自选股
  - 自选股分组管理（可创建多个分组）
  - 拖拽排序功能
  - 本地存储，浏览器关闭后保留

#### 2.2.2 组合表现概览
- 功能描述：展示用户自选股组合的整体表现
- 需求详情：
  - 组合总市值计算
  - 组合日涨跌幅
  - 组合平均股息率
  - 各股票持仓占比饼图
  - 组合收益曲线图

#### 2.2.3 盈亏计算
- 功能描述：模拟投资组合的盈亏情况
- 需求详情：
  - 手动输入持股数量、成本价
  - 自动计算浮动盈亏
  - 计算累计股息收入
  - 总投资收益率计算
  - 数据本地存储，不上传服务器

### 2.3 数据分析与图表模块

#### 2.3.1 历史股价走势图
- 功能描述：展示股票价格历史走势
- 需求详情：
  - 支持日K、周K、月K线
  - 时间范围选择：1个月、3个月、6个月、1年、3年、5年
  - 成交量叠加显示
  - 多股对比功能
  - 图表缩放、平移交互

#### 2.3.2 股息率历史趋势
- 功能描述：展示股息率历史变化
- 需求详情：
  - 股息率历史折线图
  - 股息支付时间点标记
  - 与股价走势对比图
  - 股息率与银行利率比较

#### 2.3.3 基本面指标图表
- 功能描述：可视化展示公司基本面数据
- 需求详情：
  - PE、PB、ROE等指标历史趋势
  - 同行业公司比较
  - 财务数据可视化（营收、净利润等）
  - 指标预警线设置

### 2.4 预警功能模块

#### 2.4.1 价格阈值预警
- 功能描述：股价达到设定阈值时提醒
- 需求详情：
  - 设置上涨/下跌百分比阈值
  - 设置具体价格阈值
  - 浏览器通知提醒
  - 声音提醒选项
  - 预警记录历史查看

#### 2.4.2 股息率变动提醒
- 功能描述：股息率显著变动时提醒
- 需求详情：
  - 股息率变化超过设定阈值提醒
  - 股息支付公告提醒
  - 除权除息日提醒
  - 可自定义提醒条件

#### 2.4.3 本地通知机制
- 功能描述：实现浏览器本地通知
- 需求详情：
  - 需要用户授权通知权限
  - 支持桌面通知
  - 通知历史记录
  - 可关闭特定类型通知

### 2.5 数据管理模块

#### 2.5.1 本地数据存储
- 功能描述：在用户浏览器本地存储数据
- 需求详情：
  - 使用IndexedDB存储股票数据
  - 使用localStorage存储用户设置
  - 数据加密存储（可选）
  - 存储容量管理

#### 2.5.2 数据更新机制
- 功能描述：定期更新股票数据
- 需求详情：
  - 每日自动更新一次（收盘后）
  - 手动立即更新按钮
  - 更新进度显示
  - 更新失败重试机制
  - 离线时使用缓存数据

#### 2.5.3 数据导出功能
- 功能描述：导出用户数据
- 需求详情：
  - 导出自选股列表为CSV
  - 导出投资组合数据
  - 导出预警设置
  - 数据备份与恢复功能

## 3. 非功能需求

### 3.1 性能要求
- **页面加载时间**: 首屏加载时间小于3秒
- **数据更新时间**: 批量更新100只股票数据小于30秒
- **图表渲染**: 复杂图表渲染时间小于2秒
- **本地操作响应**: 用户交互响应时间小于100毫秒

### 3.2 数据准确性
- **股价数据**: 与交易所数据延迟不超过1个交易日
- **股息数据**: 基于公司公告的准确股息信息
- **计算准确性**: 股息率等计算误差小于0.01%
- **数据一致性**: 不同页面间数据保持一致

### 3.3 用户体验要求
- **界面响应**: 所有交互都有视觉反馈
- **错误处理**: 友好的错误提示和恢复指引
- **离线体验**: 核心功能支持离线使用
- **浏览器兼容**: 支持Chrome、Firefox、Edge最新版本
- **移动端适配**: 在手机浏览器上有良好体验

### 3.4 安全性考虑
- **数据隐私**: 所有用户数据存储于本地，不上传服务器
- **API安全**: 数据API调用使用HTTPS
- **XSS防护**: 对用户输入进行严格过滤
- **本地存储安全**: 敏感数据可加密存储

## 4. 技术架构

### 4.1 前端技术栈建议
- **框架**: React 18 + TypeScript
- **状态管理**: Zustand（轻量级状态管理）
- **UI组件库**: Ant Design 或 MUI（现代设计风格）
- **图表库**: ECharts 或 Recharts
- **路由**: React Router v6
- **构建工具**: Vite
- **代码规范**: ESLint + Prettier

### 4.2 数据获取方案
- **数据源选择**:
  - 新浪财经API（免费，稳定性好）
  - 东方财富API（免费，数据全面）
  - 腾讯财经API（免费，接口简单）
- **数据代理**: 为避免CORS问题，需设置简单后端代理
- **数据缓存**: 前端实现多级缓存策略
- **更新策略**: 增量更新，减少数据流量

### 4.3 本地存储方案
- **大规模数据**: IndexedDB（存储股票历史数据）
- **用户设置**: localStorage（存储用户偏好）
- **会话数据**: sessionStorage（临时数据）
- **数据版本**: 实现数据版本管理，支持迁移

### 4.4 部署方案
- **静态托管**: Vercel、Netlify或GitHub Pages
- **代理服务**: 简单的Node.js代理服务（解决CORS）
- **域名**: 建议使用简短易记的域名
- **HTTPS**: 强制使用HTTPS
- **CDN**: 静态资源使用CDN加速

## 6. 数据源方案

### 6.1 免费数据API调研

#### 6.1.1 新浪财经API
- **地址**: `http://hq.sinajs.cn/list=sh600000`
- **数据格式**: CSV格式，包含实时股价
- **优点**: 稳定，延迟低
- **缺点**: 需要代理解决CORS

#### 6.1.2 东方财富API
- **地址**: `https://push2.eastmoney.com/api/`
- **数据格式**: JSON
- **优点**: 数据全面，有历史数据
- **缺点**: 接口较复杂

#### 6.1.3 腾讯财经API
- **地址**: `https://qt.gtimg.cn/q=sh600000`
- **数据格式**: CSV
- **优点**: 简单直接
- **缺点**: 数据字段较少

### 6.2 数据字段定义
- **基础信息**: 代码、名称、行业、地区、上市日期
- **实时数据**: 当前价、开盘价、最高价、最低价、昨收价、成交量、成交额
- **财务数据**: PE、PB、ROE、每股收益、每股净资产
- **股息数据**: 每股股息、股息率、股息支付日、除权除息日
- **历史数据**: 日K线数据（开盘、收盘、最高、最低、成交量）

### 6.3 更新频率策略
- **实时数据**: 每日收盘后更新一次
- **基本面数据**: 每季度财报发布后更新
- **股息数据**: 公司公告后及时更新
- **缓存策略**: 本地缓存7天，过期重新获取
